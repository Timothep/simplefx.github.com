Freqeuently Asked Questions

Inevitably, when learning a new data access technique (even a lightweight intuitive one like Simple.Data) there are areas that diverge other common frameworks. This tends to lead to the same question popping up, so we've grouped some answers here.

How do I see the generated provider specific operations for my Simple.Data operations?

The documentation here focusses on Sql Server for consistency, but this means we may have not documented quite how something works for a different provider. Each provider is responsible for its own syntax generation and in order to gain access to the generated querying operations (for example) in Sql Lite, you can simply attach a System.Diagnostics TraceListener. Simple.Data will output to the TraceListeners all generated provider specific syntax as it is run.

Capitalised Objects

My database schema contains a convention of all objects being in uppercase. This makes my .net code seem quite SHOUTY when I use Simple.Data! Help!

Simple.Data is able to resolve capitalised SHOUTY database schemas form the differently cased equivalents. If your database table is CUSTOMER, the following holds true:

Simple.Data:

_db.Customer.All().ToList();

Generated SQL:

    select [dbo].[CUSTOMER].[CUSTOMER_ID] from [dbo].[CUSTOMER]

Schema fever

My tables are in different schemas... how do I express the statement "select * from [foo].[bar]" where foo is a schema and bar is a table?

Simple.Data supports schemas. Try this:

Simple.Data:

_db.foo.bar.Find(_db.foo.bar.Id == 1);

Generated SQL:

@p1 = 1
select [foo].[bar].* from [foo].[bar] where [foo].[bar].[id] = @p1
